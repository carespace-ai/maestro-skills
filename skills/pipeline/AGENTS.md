# gstack Pipeline (single-issue autonomous dev sprint)

> Runs the full gstack sprint in one session — from a single GitHub issue (typically a `carespace-bug-tracker` bug) to a shipped PR. Owns the end-to-end fix loop for ONE issue against ONE repo; owns no BMAD project orchestration.

Prompt + shell skill: `SKILL.md`, `code-map.sh` (structure map so the agent reads only the
needed lines), and `CARESPACE_CONTEXT.md` (the read-once repo/architecture/HIPAA map).

## Scope

**Owns**:
- Fresh-clone-per-run discipline (all files in cwd are STALE — never trust prior state).
- Reading `CARESPACE_CONTEXT.md` once to route: identify target repo from the issue's `Repository` field, jump to the right folder via "where to look by issue type", honor HIPAA notes.
- The gstack sprint phases: Setup → Investigate → Build → Review → QA → Ship, reading the actual gstack `SKILL.md` files for methodology.
- `PIPELINE.md` as the run's working memory; anti-waste rules (never re-read / re-explore).
- Deferred tool loading via `ToolSearch` (`ENABLE_TOOL_SEARCH=true`) — load per-phase only.

**Does NOT own**: multi-story/multi-repo BMAD delivery (`oracle-pipeline`, `the-oracle-story-dev`); issue *creation* (that is `carespace-bug-tracker`); the gstack methodology itself (read from the gstack skills).

## Integration Points

### ← carespace-bug-tracker
**What's received**: a structured issue body whose `claudePrompt` was pre-generated by the tracker (Anthropic SDK). **Expectations**: treat `claudePrompt` as the starting point — don't redo root-cause analysis.

### → target repo (one of 11)
**What's passed**: a branch + a single PR fixing the issue. **Who owns lifecycle**: this skill through PR open. **Contract**: repo identified from the issue's `Repository` field matched against the context map.

## Initialization & Lifecycle

**Created by**: ClaudeHub `/pipeline` invocation with an issue. **Lifecycle**: ephemeral, single-session; working memory is `PIPELINE.md` in the fresh clone; every run is a brand-new task (no resume).

## Ownership

| Thing | Relationship | Notes |
|-------|--------------|-------|
| `PIPELINE.md` | **Owns** | The run's single source of truth / memory |
| Fix branch + PR | **Owns** | One issue → one PR |
| `CARESPACE_CONTEXT.md` | **Borrows** | Read-once reference map |
| Target repo | **Shares** | Fresh clone; ships via PR |

## State

**Holds**: `PIPELINE.md` (`## Relevant Files`, phase notes). **Mutable**: `PIPELINE.md` as phases append. **Immutable**: nothing carries across runs — cwd is treated as stale.

## Key Invariants

- **Never** trust files in the working directory from a prior run — every run is a new task; never say "already completed".
- **Never** re-read a file already read this session (context holds it) or re-explore the codebase between phases — track files in `PIPELINE.md`.
- **Must** read `CARESPACE_CONTEXT.md` exactly once at start and check HIPAA notes before touching PHI files (Profile, Client, Evaluation, Survey, …).

## Patterns

```bash
# Map structure first, then Read only the needed line range
bash code-map.sh src/components/UserDashboard.tsx   # → 91-120 export default ...
```

## Anti-patterns

```text
# ❌ Don't re-grep the same question three ways, or re-read the issue body
# ✅ One search per question; the issue body + context map were read once
```
